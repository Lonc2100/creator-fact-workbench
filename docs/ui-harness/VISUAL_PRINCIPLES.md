# UI Visual Principles

## 气质

专业，但不要老 SaaS；轻松，但不要玩具化。第一版以 Mixpost 主页的暖白、细线、平台图标和舒展留白为应用壳基调，数据页单独参考 Metabase。

## Tokens

- 背景：暖白/米白，不用冷灰工业后台。
- 文字：深棕作为主文字，暖灰作为说明文字。
- 边框：细线为主，少用重阴影。
- 圆角：8-14px，卡片不做夸张胶囊。
- 状态色：绿色成功、琥珀阻塞、玫瑰错误、蓝色信息。
- 平台色：只用于 badge/icon，不铺满大面积面板。

## 组件状态

每个核心组件至少考虑：

- default
- hover/focus
- selected/active
- disabled
- loading
- empty
- error/blocked

状态必须有文字，不只靠颜色表达。

## 可用性

- 状态可见：导入、排期、失败、阻塞、保存都要明确显示。
- 错误预防：导入先 diff preview，发布状态遵守状态机。
- 识别优于记忆：平台、状态、下一步动作都用固定 badge 和标签。
- 一致性：同一个状态在不同页面同色同文案。
