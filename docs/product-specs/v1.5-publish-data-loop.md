# V1.5: Publish and Data Recovery Loop

## Goal

Promote the workbench from a V1 mainline demo to an internal fact system for content publishing and metric recovery.

## Acceptance

- One content item can have multiple platform versions.
- Platform versions have body, script, cover note, checklist, schedule, status, failure reason, and next action.
- Calendar data is derived from platform versions.
- Metric snapshots are stored by platform version and date.
- Reviews are saved with linked actions, leads, snapshots, versions, and evidence-backed insights.
- Automation runs are recorded with trace IDs.

## Deferred

- Full drag-and-drop calendar UI.
- Live platform API publishing.
- Postgres, user accounts, permissions, and O3 observability.
