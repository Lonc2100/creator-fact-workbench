# UI-CALENDAR-METRICOOL-005：发布日历 Metricool 方向重做

## 背景

用户确认 `/` 总览首页方向正确，但 `/calendar` 仍然离 Metricool 式发布日历很远。核心问题不是颜色，而是信息结构仍像“平台行 × 日期列”的后台排班表，平台图标也只是文字缩写，缺少真实媒体识别感。

## 目标

把发布日历从平台行表格改成 Metricool 式时间轴发布画布：

`时间段行 x 日期列 -> 轻量发布卡片 -> 点击卡片滑出详情 -> 拖拽改变排期`

## 参考

- Metricool Planning：白底大画布、时间轴网格、日期列、圆角小卡片、平台图标、轻量工具条。
- 本地化参考文件：`.local/benchmarks/metricool-planificador.webp`、`.local/benchmarks/metricool-planner-dom.json`。
- 当前项目首页 UI-ART-DIRECTION-004：保持简洁、专业、创作者气质。

## 范围

- 周视图优先做到好看。
- 周视图采用 `9:00 / 13:00 / 17:00 / 21:00` 时间轴，不再使用自然堆叠卡片列。
- 平台从行维度移到筛选器和卡片首行图标。
- 卡片只保留平台图标、时间、标题、短备注；状态、检查进度、阻塞细节进入右侧详情。
- checklist、阻塞原因、下一步动作进入右侧滑出详情。
- 平台标识从单字文字改为成熟开源品牌图标路径，避免手写劣质图形。
- 卡片文字做展示层降噪，长 ID、导入噪声和测试数据不直接挤满卡片。
- 不改后端模型、API、状态机和拖拽回调边界。

## 非目标

- 不接真实平台 API。
- 不做复杂月视图深度设计。
- 不继续新增额外视觉依赖；当前平台图标统一使用 `simple-icons`。
- 不把复盘、导入、数据看板混入日历页。

## 验收

- `/calendar` 看起来是发布计划画布，而不是平台排班表。
- 周视图必须是时间轴网格，卡片落在具体时间段内。
- 卡片第一行有真实平台图标感。
- 字体排版有主次，不直接展示长串工程 ID。
- 点击卡片后出现平台版本详情抽屉。
- 拖拽卡片仍能保存排期。
- `npm run verify:harness` 与 smoke 通过。
