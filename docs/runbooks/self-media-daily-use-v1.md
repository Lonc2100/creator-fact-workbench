# Self-media Daily Use V1

这份说明给每天创作和复盘时使用。目标很简单：看数据、写内容、排期、发布交接、刷新四个平台数据、再回看数据。

## 从哪里打开

先打开本地工作台：

```text
http://localhost:3200/dashboard
```

日常只需要记住四个页面：

| 页面 | 用途 |
| --- | --- |
| `/dashboard` | 看可信数据、趋势、排行和今天是否该刷新 |
| `/content` | 写新视频想法，生成四平台稿，查看内容库 |
| `/calendar` | 看和调整未来发布排期 |
| `/import` | 刷新抖音、小红书、视频号、B站数据 |

## 早上先看什么

打开 `/dashboard` 后，先看这几件事：

- 可信内容数和指标快照是否正常。
- 最近 7 天或 30 天趋势有没有明显变化。
- 哪些内容在排行里继续表现好。
- 四个平台里哪个平台贡献了最近的数据。
- 系统提示哪些平台今天建议刷新。

这里看的是可信内容级数据，不是测试数据、后台日志或平台账号总览。

## 写新内容

进入 `/content`：

1. 写一个新视频想法。
2. 让系统生成四个平台版本：抖音、小红书、视频号、B站。
3. 检查标题、正文、标签、封面提示和平台建议。
4. 保存草稿。
5. 如果已经知道发布时间，就一起填写排期；如果还没确定，先保存草稿，之后再去 `/calendar` 排。

内容库也在 `/content`，用来找已经保存的草稿、排期内容和历史内容。

## 排期

进入 `/calendar`：

- 默认只看用户自己的未来发布排期。
- 同一个内容的四个平台版本会合并成一张更容易理解的卡片。
- 可以查看排期时间和平台版本。
- 需要调整发布时间时，从日历或内容详情进入修改。

导入的历史指标和平台数据不应该出现在默认未来排期里。

## 发布交接

当前版本不替你静默发布到真实平台。发布仍然是人工动作：

1. 从内容或日历确认要发布的版本。
2. 人工复制或交接到对应平台。
3. 发布完成后，再回系统记录或确认发布结果。
4. 发布后有新数据时，到 `/import` 刷新平台指标。

## 刷新四个平台数据

进入 `/import`，按平台处理。所有平台都是先预览，再确认保存。

| 平台 | 日常路径 | 需要用户做什么 |
| --- | --- | --- |
| 抖音 | 打开创作者中心作品详情或数据页，预览内容级指标，确认后保存 | 登录、切到正确作品页、确认保存 |
| 小红书 | 打开创作服务平台内容分析表，预览笔记数据，确认后保存 | 登录、确认表格内容、确认保存 |
| 视频号 | 打开视频号助手作品列表/数据页，扫描当前页，预览后批量确认保存 | 扫码登录、停在正确页面、确认保存 |
| B站 | 使用稿件/作品级表格或 archive/work 内容级数据，预览后保存 | 提供稳定稿件/BV 标识、确认保存 |

保存后回到 `/dashboard`，看可信数据是否更新。

## 目前不要期待什么

- 不会静默自动发布到真实平台。
- 不会全自动静默抓取四个平台。
- 不恢复公众号/WeChat 后台。
- 不把 B站账号级总览指标写进 durable totals。
- 不保存密码、cookie、token、请求头、storageState、raw 请求响应、截图、HAR、trace 或平台 DOM。
- 不把导入指标写进默认未来排期日历。

## 每天结束前

记录三件事就够：

- 今天看过的核心数据变化。
- 今天新增或调整了哪些内容和排期。
- 今天刷新了哪些平台，哪些平台还需要人工登录、扫码或补数据。
